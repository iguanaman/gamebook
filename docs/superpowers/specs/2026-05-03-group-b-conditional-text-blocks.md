# Group B: Conditional Text Blocks

## Scope

A new text block syntax that lets any block in a scene's `text` array render conditionally based on flags, stats, or visit history. Subsumes the `revisit_text` concept ‚Äî scene-memory moments become `if: visited` blocks.

---

## YAML Syntax

Any entry in the `text` array can be a conditional object instead of a plain string or voice-tagged string:

```yaml
text:
  - "You enter the cave."

  # Conditional on a flag
  - if: {flags: [perceptive]}
    text: "You notice scratches on the wall ‚Äî tool marks, deliberate."
    else: "The walls are bare stone."

  # Conditional on a stat threshold
  - if: {stats: {strength: 8}}
    text: "The boulder looks moveable."

  # Built-in visited condition
  - if: visited
    text: "The smell hits you first. You've been here before."

  # Voice-tagged block, also conditional
  - if: {flags: [knows_innkeeper]}
    male1: "Ah, you again. The usual?"

  # flags_unset condition
  - if: {flags_unset: [warned_player]}
    text: "No one has told you what waits in the dark."
```

### Condition syntax

Mirrors `requires` exactly:
- `flags: [list]` ‚Äî all listed flags must be truthy
- `flags_unset: [list]` ‚Äî all listed flags must be falsy
- `stats: {key: min}` ‚Äî stat must meet minimum
- `visited` ‚Äî built-in; true if this scene has been visited before in this playthrough

Multiple conditions can be combined:
```yaml
- if: {flags: [has_torch], stats: {perception: 6}}
  text: "By torchlight, your sharp eyes catch the inscription."
```

### `else` block

Optional. Same format as `text` ‚Äî plain string or voice-tagged. Rendered when the condition fails. If absent, the block is simply skipped.

```yaml
- if: {flags: [knows_elvish]}
  text: "The inscription reads: 'Only the worthy pass.'"
  else: "The inscription is in a language you don't recognise."
```

## Engine Changes

### State: visited scenes

Add `visited` to state shape ‚Äî a list of scene IDs (serialised as array for JSON compatibility):

```
{
  scene:   string
  stats:   { [key]: number }
  flags:   { [key]: boolean }
  visited: string[]           // NEW ‚Äî scene IDs visited this playthrough
  history: snapshot[]
}
```

`navigateTo()` adds the current scene ID to `visited` before navigating away (so `visited` contains scenes the player has already seen, not the current one). Snapshots in history include `visited` so undo restores it correctly.

### Block rendering

Current render iterates `text` blocks and outputs each as a paragraph or voice-tagged segment. New logic:

1. For each block, check if it has an `if` key
2. If no `if` ‚Äî render as now
3. If `if` present ‚Äî evaluate condition against current state:
   - `visited`: check if `state.scene` is in `state.visited`
   - Otherwise: reuse `meetsRequirements()` logic
4. Condition true ‚Üí render `text` (or voice-tagged content)
5. Condition false + `else` present ‚Üí render `else`
6. Condition false + no `else` ‚Üí skip block entirely

`visited` evaluates to `state.visited.includes(currentSceneId)` ‚Äî the scene ID being rendered, not the value of `state.scene` (which is updated before render).

Audio block indexing must account for skipped blocks ‚Äî index by rendered position, not raw array position. The `timings` array in scene YAML is regenerated by `generate_audio.py` which must also evaluate conditions (against a neutral/default state) or treat all conditional blocks as present for audio generation purposes.

---

## Audio Considerations

`generate_audio.py` generates one audio file per text block. Conditional blocks mean a scene can have different text depending on state ‚Äî audio can't be pre-generated for all combinations.

**Decision:** generate audio for all blocks unconditionally (both `text` and `else` branches get audio files). The engine plays only the audio for blocks that actually render. File naming: `{sceneId}_block_{N}_{branch}.opus` where branch is `if` or `else`.

This is a change to `generate_audio.py` and the `timings` format ‚Äî out of scope for this spec, to be addressed when audio generation is revisited. For now: conditional blocks have no audio; audio plays for non-conditional blocks only.

**`docs/storycrafting/stage-3-scene-writing.md`** ‚Äî add a section on conditional blocks:
- Use sparingly ‚Äî every branch is writing that must be worth the reader's time
- `visited` blocks are for world-memory moments ("you've been here before"), not for gating critical information
- If a conditional block is load-bearing for the plot, make sure there's always a non-conditional path that covers the same ground
